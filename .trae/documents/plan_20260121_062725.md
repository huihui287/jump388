# 修复 getTipsMoveInfo 算法：基于几何阵型分析

## 目标
摒弃 `canSwapFormMatch` 的模拟交换逻辑，改为直接分析传入的 3 个格子（`group`）的空间几何关系，精准找出需要移动的格子和方向。

## 算法逻辑

1. **预处理**
   - 提取 `group` 中的 3 个 `gridCmpt` 组件。
   - 对它们进行排序（先按行 `h`，再按列 `v`），便于后续计算相对位置。

2. **阵型识别**
   我们将识别以下两种核心阵型：

   **模式 A：两连珠 (XX)**
   - 在 3 个格子中，找出两个坐标相邻的格子（`G1`, `G2`）。
   - 它们确定了一个轴向（横或纵）。
   - 此时，这两个格子两端的延伸位置即为 **目标空位**。
   - 检查第 3 个格子（`G3`）是否在某个目标空位的旁边。
   - 如果是，则 `G3` 是移动者，方向指向那个空位。

   **模式 B：夹心 (X_X)**
   - 在 3 个格子中，找出两个同轴但中间隔了一个格子的对子（`G1`, `G2`）。
   - 中间那个位置即为 **目标空位**。
   - 检查第 3 个格子（`G3`）是否在目标空位的旁边。
   - 如果是，则 `G3` 是移动者，方向指向那个空位。

   **模式 C：L型/T型 (潜在)**
   - 如果上述两种直线型都不满足（虽然 checkTipsGroup 通常返回直线型潜力股），则可能是一个拐角结构。
   - 此时通常任意移动一个能凑成直线的即可，但根据用户描述，主要处理上述两种直线归位的情况。

3. **执行计算**
   - 遍历 `group` 中的每一对格子，计算距离。
   - 距离=1：记录为相邻对，推算两端空位。
   - 距离=2（同轴）：记录为夹心对，推算中间空位。
   - 找到空位后，看剩下的那个格子是否与空位相邻。
   - 输出：`{ grid: 剩下的格子, dir: 指向空位的方向 }`。

## 验证
- 模拟 `X X O X`（O为异色）：识别出两个 `X` 相邻，第三个 `X` 在 O 的旁边。输出第三个 `X` 移向 O。
- 模拟 `X O X`（O为异色）：识别出两个 `X` 隔空，第三个 `X` 在 O 的旁边（上下）。输出第三个 `X` 移向 O。
