## 修正方案

1. **重构 `getGroupedGridData` (Turret.ts)**:
   - 移除 `sort` 逻辑。
   - 采用保持顺序的归类方式：遍历 `gridDataList` 时，按照类型出现的先后顺序存入结果数组。

2. **修正 `updateCapacityView` (Turret.ts)**:
   - 采用倒序遍历 `children`（从底部到顶部）。
   - 数据指针从 `0` 开始（即将发射的种类）。
   - 逻辑：
     - `i` 从 `children.length - 1` 递减到 `0`。
     - 如果 `dataIndex < gridData.length`，则显示该节点并设置数据，`dataIndex` 递增。
     - 否则，隐藏该节点。
   - 这样能保证：**即将发射的种类在最底部，数据不足时优先隐藏顶部的节点。**

## 代码变更预期

### getGroupedGridData
不再使用 `summary.forEach` + `sort`，改为遍历时直接维护一个有序数组，确保 `result[0]` 是 `gridDataList[0]` 的类型。

### updateCapacityView
```typescript
let dataIndex = 0;
for (let i = children.length - 1; i >= 0; i--) {
    if (dataIndex < gridData.length) {
        // 设置 child[i] = data[dataIndex]
        dataIndex++;
    } else {
        // 隐藏前面的节点
    }
}
```
